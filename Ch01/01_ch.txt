# 1장 들어가기
## 1. 구성
* 11개의 장(주제), 90개의 아이템(규칙)으로 구성
* 주제 : 넓은 관점에서 소프트웨어 설계의 한 측면
* 규칙 : 업계 최고의 베테랑 프로그래머들이 유익하다고 인정하는 관례
* 아이템은 서로 독립적이라 순서대로 읽을 필요 없음
## 2. 특징
* 규칙 대부분은 아주 핵심적인 기본 원칙인 명료성과 단순성에서 파생
* 이러한 규칙을 생각없이 맹정하지는 말아야하지만, 어겨야할때는 합당한 이유가 필요
* 정석이 되는 규칙을 배우고, 언제 그 규칙을 깨도 되는지 익혀야함
## 3. 성능
* 성능에 집중하는 대신, 명확, 정확, 유용, 견고, 유연, 관리하기 쉬운 코드를 작성하는데에 집중
* 그래도 대부분 상황에서 원하는 성능에 도달 가능
## 4. 용어
* 대부분 Java 언어 명세에 따름
* 예외
 * 상속 -> 서브클래싱
 * 인터페이스 상속 -> 구현, 확장
 * 패키지 접근(아무것도 명시하지 않은 접근 수준) -> 패키지-프라이빗
 * Java 명세에 없는 것들
  * 공개 API, API 사용자, API 클라이언트, API 요소 등
